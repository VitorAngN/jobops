- [Backlog priorizado](10-backlog-priorizado.md): o que falta, impacto e ordem sugerida.

## Validacao

- [Evidencias tecnicas](07-evidencias-tecnicas.md): comandos, prints e sinais de funcionamento.
- [Memoria de projeto](11-memoria-de-projeto.md): resumo para retomar contexto rapido.

## Como manter estes documentos

- Atualize a memoria quando uma decisao importante for tomada.
- Atualize o backlog quando uma tarefa sair de "pendente" para "feito".
- Atualize evidencias sempre que houver build, teste, CI ou deploy validado.
- Prefira documentos pequenos e linkados em vez de um arquivo gigante.
